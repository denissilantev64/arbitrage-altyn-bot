# Project instructions

- The runtime is Python 3.11+ with aiogram long polling, aiohttp and SQLite. Run checks with `.venv/Scripts/python -m pytest`, `.venv/Scripts/ruff check .`, `.venv/Scripts/ruff format --check .` and `.venv/Scripts/mypy src` on Windows; use the corresponding `.venv/bin/*` paths on Linux.
- Keep every price and fee calculation in `Decimal`. Altyn is a purchase-only, amount-specific quote from private `GET /website/arbitrage-rate/`; `ALTYN_ARBITRAGE_TOKEN` is required, secret, and must exist only in the production env. The returned rate already includes Altyn's personal commission, and its fixed network fee applies only to `/spread n`. Do not add separate Altyn fee env values or restore the unsupported `Rapira -> Altyn` direction. Scheduled snapshots use 1,000,000 RUB once per minute; accept them for at most two minutes. Exact `/spread n` quotes are fetched on demand and cached in memory for 15 seconds per amount. Start uncached Altyn requests at least 10 seconds apart: reject an on-demand request when the local slot is busy, but let the scheduled refresh wait for the slot. Rapira bid/ask come from the current public order book, while the taker fee comes dynamically from `market/fee/page-query`; do not replace any missing field with zero or a last-trade price. Keep morning-broadcast retries on a separate one-minute interval.
- The support UI is intentionally disabled. Do not restore a support button, support handler, or `SUPPORT_URL` runtime setting unless the product requirement explicitly enables it again.
- SQLite schema changes require an explicit `PRAGMA user_version` migration. Schema version 2 archives incompatible version-1 rates in `rate_snapshots_v1_archive` and `rate_refresh_state_v1_archive`; never silently recreate, reinterpret, or use those archived rows as current production rates.
- Production runs as the `arbitrage-bot` systemd user. Code is under `/opt/arbitrage-altyn-bot`, mutable data under `/var/lib/arbitrage-altyn-bot`, and runtime secrets under `/etc/arbitrage-altyn-bot/bot.env`.
